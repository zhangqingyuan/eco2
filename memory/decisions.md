## 决策列表

### D20260621-001：文件为记忆真实来源
- **日期**：2026-06-21
- **类别**：架构设计
- **决定者**：Copilot Agent + User
- **背景**：
  - 跨对话的连贯性是 eco2 工作空间的核心需求
  - GitHub Copilot 网页版每次对话无法直接获取前一次对话的上下文
- **选项**：
  1. 完全依赖对话历史（不可行，对话结束后消失）
  2. 用外部 API 存储（复杂度高，超出个人工作空间范围）
  3. **选定：用仓库文件作为唯一真实来源** ✓
- **决策内容**：
  - 所有长期信息必须保存到 AGENTS.md、MEMORY.md、memory/ 中
  - 对话内容视为临时上下文，仅用于当前任务
  - 每次任务完成后立即更新相关文件
- **影响**：
  - 强制性的记忆更新流程（必做）
  - 更高的信息可靠性和跨对话连贯性
  - 维护成本略升（但可接受）
- **状态**：已执行 ✓

### D20260621-002：轻量级记忆系统设计
- **日期**：2026-06-21
- **类别**：系统设计
- **决定者**：Copilot Agent + User
- **背景**：
  - 参考 OpenClaw 的理念，需要一个"能做事"的持久化 AI 助手
  - 但不希望过度设计导致维护成本过高而放弃
- **选项**：
  1. 复杂的多层级知识图谱（Neo4j 或类似）
  2. 数据库 + Web UI（完整但重型）
  3. **选定：文件系统 + Markdown（轻量级）** ✓
- **决策内容**：
  - 核心文件：AGENTS.md（角色）+ MEMORY.md（进度）
  - 扩展文件：memory/ 子目录（goals, projects, skills, decisions）
  - 可选文件：SOUL.md（个性），logs/（日志）
  - 文件格式：纯 Markdown，易于编辑和版本控制
- **影响**：
  - 低维护成本，易于持续使用
  - 完整的 Git 历史和版本追踪
  - 支持与其他人协作（如需要）
- **状态**：已执行 ✓

### D20260621-003：memory/ 目录结构
- **日期**：2026-06-21
- **类别**：架构设计
- **决定者**：Copilot Agent + User
- **背景**：
  - 需要将 MEMORY.md 中的细节分散到多个专用文件
  - 便于分类和快速查找
- **选项**：
  1. 将所有信息放在 MEMORY.md（过长，难以维护）
  2. **选定：四个子文件** ✓
     - `goals.md` - 长期目标与里程碑
     - `projects.md` - 活跃项目与进度
     - `skills.md` - 技能、工具、知识库
     - `decisions.md` - 决策记录（本文件）
- **决策内容**：
  - MEMORY.md 作为"导航中枢"，包含指向子文件的链接
  - 每个子文件独立维护，模块化清晰
- **影响**：
  - 信息组织更清晰
  - 易于按需更新和查询
- **状态**：已执行 ✓
